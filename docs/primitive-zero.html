Primitive Zero

La capa más baja del design system: cada etiqueta HTML semántica estilizada globalmente con tokens, sin necesidad de clases.

Un <blockquote> se ve bien porque RenDS lo viste. Un <kbd> tiene su identidad tipográfica. Un <details> funciona como accordion básico. El diseñador debe conocer estas etiquetas como vocabulario base — RenDS garantiza que cada una tenga tipografía, color, forma y medidas accesibles por defecto.

¿Cuándo una etiqueta "sube" de Primitive Zero a Primitive (con clase)?

Cuando necesita una o más de estas cosas: (a) variantes — un <button> que también sea ghost, secondary, danger; (b) sub-elementos — un Card con header, body y footer; (c) API de tokens pública — --ren-{componente}-*; (d) JavaScript para accesibilidad — focus trap, roving tabindex, coordinación entre items.

Si la etiqueta HTML nativa + classless es suficiente, se queda acá.

Estructura

Los elementos de landmark (<header>, <nav>, <main>, <article>, <section>, <aside>, <footer>) son contenedores semánticos puros. Primitive Zero no les aplica estilo visual (background, border, padding) — sólo reset de márgenes y flujo. El diseñador los usa como anclas para skip links, accesibilidad con screen readers, y scoping de estilos.

Landmarks semánticos

Contenedores sin estilo visual. Estructura pura.

<header> — Cabecera de la página o sección
<nav> — Menú de navegación principal
<main> — Contenido principal único (uno por página)
<article> — Contenido auto-contenido (post, card)
<section> — Agrupación temática con heading
<aside> — Contenido relacionado pero secundario
<footer> — Pie de página o sección

Combinaciones contextuales

Primitive Zero no se limita a estilar etiquetas individuales — también define el comportamiento visual de combinaciones semánticas comunes. Un <cite> dentro de un <blockquote> cambia su rol (firma) y por ende su forma (block con em-dash). Estas reglas viven en classless.css y se activan automáticamente.

<blockquote> > <cite>

Cita dentro de blockquote = firma. Block + em-dash prefix.

Design is not just what it looks like and feels like. Design is how it works.

Steve Jobs, The New York Times, 2003 
<blockquote>
  <p>…</p>
  <cite>Steve Jobs</cite>  /* firma automática */
</blockquote>

/* vs. cite standalone */
